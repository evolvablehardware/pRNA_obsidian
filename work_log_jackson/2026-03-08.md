## Log
- Multiple circuits on one fpga
	- I saw direct routing version of bitstreamevolution Thomas is working on a while back and I think implements everything that is required for this, as the representation is not specific to tiles (aside from the few different logic tiles). It also only uses the neigh nets, so the global and large spans are not an issue. All that needs to be done is to splice the circuits onto different locations and include multiple io routing pins in the seed.
	- All I did was change the tile locations before writing the circuit
		- Two different genomes spliced onto the same circuit, https://github.com/heiljj/BitstreamEvolution/blob/pico-ice-cgp/different_genome_example.asc

	- Once the direct routing representation is complete, it should be incredibly simple to add support for multiple circuits. From the icefarm side of things, this would mean adding an additional pin to experiment firmware and reporting those results in addition. This will work especially well with the pulse count experiments, as the pulses are counted async with PIO. Not sure how well it will work for varmax since it uses polling.
